# Code Review

Please perform a comprehensive code review of the selected code or entire file.

Review for:
1. **Code Quality**: Clean code principles, readability, maintainability
2. **Security**: Potential vulnerabilities (SQL injection, XSS, auth issues, etc.)
3. **Performance**: Inefficient algorithms, unnecessary operations, memory leaks
4. **Best Practices**: Language-specific idioms, design patterns, SOLID principles
5. **Testing**: Test coverage, edge cases, error handling
6. **Documentation**: Code comments, API docs, unclear logic

Provide:
- Specific issues found (with line numbers if possible)
- Severity rating (Critical/High/Medium/Low)
- Concrete suggestions for improvement
- Code examples showing recommended changes
- Explanation of WHY each change improves the code

Be constructive and educational in your feedback.
